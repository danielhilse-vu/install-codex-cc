## Design Context

### Users
Internal software engineers setting up Codex + Claude Code on remote Coder workspaces. They are technical, context-switched, and arriving with a task to complete — not browsing. They trust tooling that looks like it was built with the same care as the tools it documents.

### Brand Personality
Precise, trustworthy, minimal. Not a marketing page. Not a consumer app. A well-made internal artifact that respects the reader's time.

### Aesthetic Direction
**Precision Technical** — editorial light mode with always-dark code blocks. Clean warm-tinted backgrounds (not pure white). Strong typographic hierarchy. Semantic color: amber for shell/terminal commands, violet for Claude Code prompts. No decorative effects, no glassmorphism, no gradient text. The design should feel like it was written by someone who cares about the craft.

Anti-references: generic AI-generated docs (cyan-on-dark, purple-blue gradients, cards with icons above every heading, left-border callouts as decoration).

### Design Principles
1. **Semantic color, not decorative color** — amber means terminal, violet means Claude Code. Color carries information.
2. **Hierarchy through contrast** — make big things bigger, small things smaller. Not everything should be the same size.
3. **Code blocks as first-class citizens** — always dark, always distinctive, always instantly scannable.
4. **No noise** — every element must earn its place. Internal tools respect the reader.
5. **Warm, not cold** — slightly warm tints on backgrounds and text instead of pure black/white. Makes long reading sessions more comfortable.
